# OpenRoot Contributing

All work scored with CoE process before delivery.

## Data & Instrumentation Rule (Community Burden)
Founder/manual sensory input is eliminated for Node Zero and all official examples.  
The open-source community carries the data burden by adopting and running their own loggers on replicated Nodes, then contributing verifiable CSV/IPFS data back. This enables collective PoPW verification and accelerates adoption.

## How to Contribute Sensor Data
1. Deploy the community logger on your Node (see scripts/community-logger/).
2. Run it during your cure or operational phase.
3. Commit CSV or IPFS hash + short validation note via PR or bounty claim.
4. Highest CoE goes to clean, replicable, poorest-community-friendly contributions.

See docs/thermal-loop-modularity.md for panels that can run standalone.
